An AI-powered web application for real-time wildfire risk prediction, 24-hour forecasting, and email alerting вЂ” built with Flask, scikit-learn, XGBoost, and MapLibre GL.

**Live demo:** https://codecurators-t08.onrender.com

---

## Features

- **Interactive map** вЂ” click any location to run a prediction
- **Fire risk classification** вЂ” ensemble VotingClassifier (XGBoost + Random Forest + Logistic Regression, 3Г— balancing strategies)
- **Fire Radiative Power (FRP) estimation** вЂ” XGBoost regression model
- **Live weather** вЂ” pulled from Open-Meteo at prediction time
- **24-hour forecast** вЂ” runs both models on each upcoming hour
- **Explainable AI** вЂ” horizontal bar charts showing which features drove the prediction
- **Email alerts** вЂ” optional, per-user, with configurable risk threshold
- **Prediction history** вЂ” table + trend chart of all past predictions
- **OTP-verified signup** вЂ” 6-digit code sent to email, expires in 5 minutes
- **Rate limiting** вЂ” per-route limits on predict, forecast, explain, and OTP endpoints

## Setup

### 1. Clone the repo

```bash
git clone https://github.com/yourusername/wildfire-portal.git
cd wildfire-portal
```

### 2. Create a virtual environment

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Mac/Linux
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Set up your `.env` file

Copy the example and fill in your values:

```bash
cp .env.example .env
```

Then edit `.env`:

```env
SECRET_KEY=your-long-random-secret-key
GMAIL_USER=wildfirepredictor@gmail.com
GMAIL_APP_PASSWORD=your-16-char-app-password
DB_PATH=users.db
FLASK_DEBUG=false
```

To generate a secure `SECRET_KEY`:

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

To get a Gmail App Password:
1. Go to [myaccount.google.com](https://myaccount.google.com) в†’ Security
2. Enable 2-Step Verification if not already on
3. Search for "App Passwords" в†’ generate one for Mail
4. Paste the 16-character password into `.env`

### 5. Place your model files

Put these two files in the project root (same folder as `app.py`):

```
voting_ensemble_fire.joblib
fire_regression_model_2.pkl
```

### 6. Run the app

```bash
python app.py
```

Then open [http://127.0.0.1:5000](http://127.0.0.1:5000) in your browser.

> вљ пёЏ Always open the app via `localhost` вЂ” never open `.html` files directly in the browser, as Jinja2 templates will not render correctly.

## Dependencies

Install all at once with:

```bash
pip install -r requirements.txt
```

**Core:**
- `flask` вЂ” web framework
- `flask-cors` вЂ” cross-origin support
- `flask-limiter` вЂ” rate limiting
- `python-dotenv` вЂ” loads `.env` file
- `werkzeug` вЂ” password hashing (included with Flask)

**ML:**
- `joblib` вЂ” model loading
- `scikit-learn==1.6.1` вЂ” VotingClassifier, RandomForest, LogisticRegression, StandardScaler
- `xgboost` вЂ” XGBClassifier and XGBRegressor

**Data & API:**
- `numpy` вЂ” array operations
- `requests` вЂ” Open-Meteo weather API calls

> вљ пёЏ The models were trained on scikit-learn `1.6.1`. Using a newer version (like `1.7.x`) will show `InconsistentVersionWarning` on load. Everything will still work, but ask your teammate to retrain and re-export the models when possible.

## Notes

- **No data is ever sent to a third party** other than Open-Meteo (weather) and MapTiler (map tiles). Both are read-only API calls with no user data attached.
- **The OTP is never sent to the browser.** It is generated server-side, stored in the session, and emailed directly.
- **Alert emails are only sent when:** alerts are enabled by the user, and the prediction risk meets or exceeds their chosen threshold.
- **The SQLite database** (`users.db`) is auto-created on first run. For production, consider migrating to PostgreSQL.

## Deployment (Render.com вЂ” Free Tier)

1. Push the project to a GitHub repo (make sure `.env` and `users.db` are in `.gitignore`)
2. Go to [render.com](https://render.com) в†’ New в†’ Web Service
3. Connect your GitHub repo
4. Set:
   - **Build command:** `pip install -r requirements.txt`
   - **Start command:** `gunicorn app:app`
5. Add all your `.env` variables under **Environment в†’ Add Environment Variable**
6. Add `gunicorn` to `requirements.txt`
7. Deploy

> The free tier spins down after 15 minutes of inactivity вЂ” first load after that takes ~30 seconds.
